# 设计规范 — Prompt Recorder

## 设计目标

简洁、克制、温暖 — 参考 Claude 的设计语言，减少视觉噪音，让用户聚焦于内容本身。

## 配色方案

| 色值 | 用途 |
|-------|------|
| `#D97757` | 主色调 — 按钮、链接、选中态、分类标签 |
| `#C56545` | 主色调 hover — 按钮悬停 |
| `#FAFAF9` | 全局背景 — 暖白色 |
| `#FFFFFF` | 卡片/输入框背景 |
| `#1A1A1A` | 主文字色 |
| `#6B6B6B` | 次要文字色 — 时间戳、辅助信息 |
| `#E8E4DF` | 边框色 — 柔和分隔 |
| `#F5F2EF` | 浅色强调背景 — 列表 hover、面板底色 |

## 字体

- 默认系统字体（Windows: Microsoft YaHei 或 Segoe UI）
- 输入区：等宽字体（Consolas / Cascadia Code）
- 正文大小：14px，标题：16px，辅助文字：12px

## 布局

三栏水平布局，使用 QSplitter 实现可拖拽调整：

```
┌──────────────────────────────────────────────────────┐
│  Prompt Recorder                         [⚙ 设置]    │
├─────────────────┬─────────────────┬──────────────────┤
│                 │                 │    🔍 搜索...    │
│   输入区域      │   分析结果      │   ─────────────  │
│                 │                 │   [标签] 标题    │
│   (大文本框)    │   原文          │   2024-01-01     │
│                 │   ────────     │   ─────────────  │
│                 │   优化版        │   [标签] 标题    │
│                 │                 │   2024-01-01     │
│                 │   [分类标签]    │                  │
│                 │   ℹ 优化说明   │                  │
│                 │                 │                  │
│   [分析]        │ [采用] [保留]  │                  │
└─────────────────┴─────────────────┴──────────────────┘
```

- 默认窗口大小：1200 × 750
- 最小窗口：900 × 500
- 三栏默认比例 1:1:1
- 圆角：4px（输入框、按钮、卡片）

## 交互规范

- 按钮状态：默认 / hover（加深）/ disabled（灰色）
- 加载状态：按钮文字变为「分析中...」+ 禁用按钮
- 错误处理：在结果区显示错误信息，红色文字
- 确认反馈：操作成功后在左下角短暂显示 toast 提示

## 按钮样式

```
主按钮：背景 #D97757，文字白色，圆角 4px，padding 8px 24px
次按钮：背景透明，边框 #E8E4DF，文字 #6B6B6B
``` 

## 组件间距

- 面板内边距：16px
- 元素间距：8px（紧凑）/ 16px（标准）
- 面板间距由 QSplitter 的分割线控制
